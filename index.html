DooVR

DooVR is a 3D modelling application that lets users create 3D models in Virtual Reality. A more detailed description of the project can be found here.

Throughout this project I have had a leading role. I have worked with on datastructures, rendering, interface, user testing, calibration and synchronisation of tracking devices, integration of a lot of different APIs and much more.

SwordSwing

This is a game I'm making in Unreal Engine. I'm trying to make a multiplayer melee fighter where everything is based on physics. No keyframes and no "setWorldPosition()"s only forces, impulses and automatic control. The basic gameplay functionality is implemented and the weapon handling is working somewhat as intended. Before I consider myself done with this project I want to make the weapon handling more dynamic and robust, implement physical biped walking in some capacity and create some solution for synchronizing physics states across several clients.

In this project I have worked with unreal engine, character design, 3D modelling, gameplay programming, animation, particle effects, automatic control, procedural materials and some procedural terrain generation.



Volumetric mesh breaking

This is a project I worked on as part of the doctorate course TN1008 at Linköpings University. The basics of the algorithm i use is describe here. The main algorithm is implemented and working but I still have a lot of optimizing to do before it can be used in a runtime project. Specifically I want to calculate almost everything on the GPU since the algorithm is well suited for GPU calculations. Right now everything is done on a single CPU thread. I also want to explore more partial breaking i.e. not breaking the entire mesh only parts of it.



3D Tower Defense

In this project I have worked with character design, 3D modelling, gameplay programming, animation and particle effects.



Bsplines - adaptive sampling

This is a project I made in the course DM6122 at Nanyang Technological University. It is a Bspline editor written in c++ using opengl. It uses both uniform and adaptive sampling.

This project was mostly opengl programming and spline math. I also used glut for window handling and input for the first time. I did it because our proffesor was using it and I had never tried it before. It worked ok but I vastly prefer GLFW.



MCGPU

MCGPU is a project that me and Isabelle Forsman is currently working on. In it we procedurally generate terrain. It is done by procedurally generating a scalar field and then running the marching cube algorithm on it. Both the scalar field generation and marching cube algorithm runs in openGL (on the gpu)

In this project I have worked with on datastructures, rendering and procedural algorithms.
